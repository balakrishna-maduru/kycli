## Reporting a Vulnerability

We take the security of KyCLI seriously. If you discover a security vulnerability, please follow these steps to report it responsibly:

1.  **Do NOT** open a public issue on GitHub.
2.  Email the vulnerability details to **balakrishnamaduru@gmail.com**.
3.  Include as much information as possible:
    *   Description of the vulnerability.
    *   Steps to reproduce.
    *   Potential impact.
    *   Any mitigations you have already identified.

### Response Timeline

*   We will acknowledge receipt of your report within **48 hours**.
*   We will provide a timeline for investigation and fix within **1 week**.
*   Once fixed, we will release a patch version and credit you (if desired) in the release notes.
